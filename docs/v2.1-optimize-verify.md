# v2.1 优化验证手册

本文件提供**一次性本地验证**步骤，用于确认：

- CF Worker 修改后的代码能正常构建
- EdgeOne 路径代码能正常运行
- 优化效果可被观测
- 新旧版本可直观对比

---

## 1. CF Worker 构建验证

```bash
cd packages/cf-worker
node build.mjs
```

预期输出：
```
✓ Built dist/worker.js (version: <git-short-hash>)
```

验证构建后的文件包含新功能：

```bash
grep -c "cacheTtl: 86400" dist/worker.js     # 应输出 1
grep -c "content-length" dist/worker.js       # 应输出多个（Range + 两处 delete）
grep -c "Crawl-delay: 10" dist/worker.js      # 应输出 1
```

## 2. content-length 错误验证（修复项）

**问题回顾**：Cloudflare Worker 日志中 `Provided readable stream must have a known length` 错误。

**本地静态检查**：确认两处 `content-length` 删除已存在：

```bash
# /_cdn/ 分支：upstreamResp 的 content-length 被删除
grep -n "newHeaders.delete.*content-length" worker.js
# 应有 1 行输出（位于 /_cdn/ 分支）

# HTML 分支：originResp 的 content-length 被删除
grep -n "respHeaders.delete.*content-length" worker.js
# 应有 1 行输出（位于 HTML 处理分支）
```

**上线后验证**（可选）：

```bash
# 部署后观察 Worker 日志
wrangler tail | grep "known length"
# 如无输出说明修复生效
```

## 3. 缓存优化验证（减少 Worker 调用）

**问题回顾**：爬虫每次请求都触发 Worker 调用，消耗免费额度（100k/天）。

**本地静态检查**：

```bash
# 确认 /_cdn/ 上游 fetch 有 cacheTtl: 86400（24 小时边缘缓存）
grep "cacheTtl: 86400" worker.js
```

**缓存层级说明**：

| 层级 | 位置 | TTL | 命中效果 |
|------|------|-----|----------|
| Edge Cache | Cloudflare 边缘节点 | 24h | 完全绕过 Worker，零调用 |
| R2 | 持久化存储 | 永久 | 边缘未命中时从 R2 读取，一次 Worker 调用 |
| Origin | Webflow 源站 | — | 仅首次或 R2 未命中时回源 |

**上线后效果**：
- 单页面加载：原先 7-15 次 Worker 调用 → 2-4 次（HTML + 首次未缓存资源）
- 爬虫访问：首次后全由边缘缓存响应，不再触发 Worker

## 6. 新旧版本对比方法

### 使用 git diff 查看代码变更

```bash
# 与 v2.0-geo-fix 分支对比（上一个稳定版本）
git diff v2.0-geo-fix...optimize/cf-quota-edgeone-cleanup -- packages/cf-worker/
```

### 主要变更清单

| 变更 | 文件 | 说明 |
|------|------|------|
| cacheTtl: 86400 | worker.js /_cdn/ fetch | 24h 边缘缓存，减少 Worker 调用 |
| content-length 删除（/_cdn/） | worker.js newHeaders | 修复 ReadableStream body 错误 |
| content-length 删除（HTML） | worker.js respHeaders | 修复 HTMLRewriter 后 body 错误 |
| Crawl-delay: 10 | worker.js robots.txt | 降低爬虫访问频率 |
| EdgeOne 目录合并 | packages/edgeone/ | 删除 duplicate，保留 v2.0 结构 |
| EdgeOne README 更新 | packages/edgeone/README.md | 合并 v1→v2 文档 + 新功能说明 |

### 直接查看当前分支相比 main 的更改

```bash
git log --oneline v2.0-geo-fix..optimize/cf-quota-edgeone-cleanup
```
